# Repo Research

Understand the repo before extracting ideas.

Inspect:

- README and install docs.
- Folder structure and entrypoints.
- Examples, tests, or sample configs.
- Release/activity signals when relevant.
- License and usage constraints.

Separate what the repo actually supports from what it implies.
