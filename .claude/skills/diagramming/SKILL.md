# Diagramming

Use diagrams to clarify relationships, not decorate docs.

Choose:

- Flowchart for process.
- Sequence diagram for interactions over time.
- System diagram for services and boundaries.
- State diagram for lifecycle rules.

Keep labels short and concrete. Verify rendered output when possible.
